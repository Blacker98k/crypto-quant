# AGENTS.md · 给 AI Agent 的项目入门指南

> **任何 AI Agent 进入此项目时，第一件事必须读完本文档**。
> 本项目对 Agent 的协作约束有具体规定，违反约束的提交不会被合入。

---

## 1. 你是谁、要做什么

本项目（crypto-quant）是**多策略虚拟币量化交易系统**。维护者 Blacker 通过多个 AI Agent 接力开发——你可能是：

- **会话型 Agent**（Cursor / Claude / GPT 等内嵌在 IDE）：每次会话临时启动，没有持久化身份
- **常驻 Agent**（OpenClaw 这类 7×24 在线工具）：独立部署在维护者本机，有完整 shell 能力

无论哪种，你的产出都必须经维护者审阅才能合入主分支。详见 [`docs/07-决策日志/ADR-009-远程Agent与可信代码边界.md`](./docs/07-决策日志/ADR-009-远程Agent与可信代码边界.md)。

## 3. 协作边界（不可妥协）

完整规则见 [ADR-009](./docs/07-决策日志/ADR-009-远程Agent与可信代码边界.md)。

### 3.1 你能做什么

- ✅ 读项目里**任何文件**
- ✅ 在本地工作目录写文件、改文件、跑命令
- ✅ 在本地生成 patch / diff
- ✅ 提 PR 到 `agent/<你的名字>/<task>` 或 `feat/*` `fix/*` 分支

### 3.2 你不能做什么

- ❌ **直接 push 到 `main`** —— 分支保护强制阻止
- ❌ **修改高敏感文件**（见 §3.3）
- ❌ **跳过 PR 流程**（哪怕是"看起来很小的改动"）
- ❌ **跳过 CI 检查**（CI 红的 PR 不能合）
- ❌ **修改 ADR**（修改决策必须维护者本人）

### 3.3 高敏感文件白名单（绝对禁止 Agent 直接改）

| 路径 | 内容 | 原因 |
|---|---|---|
| `config/secrets.yml` `.env` `*.key` `*.pem` | 凭据 / API key | 资金安全 |
| `config/risk.yml` | 风控阈值（仓位 / 杠杆 / 熔断） | 可能让风控失效 |
| `config/runtime.yml` | 实盘开关、二阶段确认 | 直接关乎实盘 |
| `.gitignore` 里"会改变敏感数据入库范围"的条目 | secrets / data 范围 | 可能让 secrets 进 git |
| `scripts/deploy.sh` `systemd/*` | 部署脚本 | 可能让生产挂掉 |
| `docs/07-决策日志/ADR-*.md` | 已生效的决策 | 推翻决策必须维护者本人 |

如果你的任务**必须**修改这些文件之一：
- 在 PR 描述里**显式声明**你想改什么 + 为什么
- **不要自己改**，等维护者本人接手

`.gitignore` 例外：见 [ADR-009 §2.3.1](./docs/07-决策日志/ADR-009-远程Agent与可信代码边界.md)。

## 4. 分支命名规范

| 前缀 | 用途 | 例子 |
|---|---|---|
| `feat/` | 新功能 | `feat/p1-data-feed-binance` |
| `fix/` | bug 修复 | `fix/oms-stop-loss-timeout` |
| `chore/` | 杂项（CI / 工具 / 配置） | `chore/governance-setup` |
| `docs/` | 纯文档改动 | `docs/p1-data-layer-detail` |
| `agent/<your-name>/` | Agent 专用工作分支 | `agent/openclaw/p2-factor-lib` |
| `experiment/` | 探索性，可能丢弃 | `experiment/llm-news-sentiment` |

**main 分支永远只接受 PR 合并的 commit，从不直推**——分支保护规则强制阻止直推。

`feat/` `fix/` 前缀建议带 phase 标识（`p1-` `p2-`），便于路线图回溯。

## 5. Commit Message 规范（Conventional Commits）

格式：

```
<type>(<scope>): <subject>

<body>

<footer>
```

| type | 用途 |
|---|---|
| `feat` | 新功能 |
| `fix` | 修 bug |
| `docs` | 仅文档 |
| `refactor` | 重构（行为不变） |
| `test` | 增改测试 |
| `chore` | 工具 / CI / 配置 |
| `perf` | 性能优化 |

**Agent 提交必须额外带 `[agent: <name>]` 前缀，且 footer 标注 session / reviewer**：

```
[agent: openclaw] feat(strategy): 实现 ATR-based position sizer

按 docs/03-详细设计/03-风控层.md §4 实现 sizer。
单测覆盖 5 类边界。

Generated by: openclaw
Session: 2026-05-15-14:23
Reviewed by: pending
```

会话型 Agent 也建议在 commit 里说明任务来源（哪条用户消息触发的）。

## 6. 完成任务必做

每个 PR 必须通过下面的 checklist（CI 部分自动检查、其他人工审）：

- [ ] **代码改动有单测覆盖**（除非纯文档/CI/工具）
- [ ] **接口改动同步更新对应文档**——CI 自动检查（见 §7）
- [ ] **风控相关改动**有 ADR 链接或新写 ADR
- [ ] **会话末尾更新** [`记忆/02-当前状态.md`](./记忆/02-当前状态.md)
- [ ] **PR 描述**填完所有字段（PR 模板会自动展开）
- [ ] **没有任何 secrets / API key** 进入 git diff

## 7. 文档与代码同步规则（CI 自动检查）

CI 会扫 PR diff，触发以下规则会让 PR 红、不能合：

| 改了这些代码 | 必须同步改这些文档 |
|---|---|
| `core/strategy/**/*.py` | `docs/04-接口文档/01-策略接口.md` |
| `core/data/**/*.py` | `docs/04-接口文档/02-数据接口.md` |
| `core/execution/**/*.py` | `docs/04-接口文档/03-订单接口.md` 或 `docs/03-详细设计/04-执行层.md` |
| `core/risk/**/*.py` | `docs/03-详细设计/03-风控层.md` |

如果你确实**只改了实现而接口/契约不变**，可以在 PR 描述里加一行 `Skip-Doc-Sync: 仅实现细节调整，契约未变` 跳过此检查。
