# 01 · 策略接口

> **稳定性**：**Beta v0.1**（Phase 3 实盘前可破坏式变更；Phase 4 上线前冻结为 Stable v1.0）
> **用途**：定义所有策略必须遵守的统一契约——研究和生产共用同一份代码不改一行。
> **关联**：[架构 §3.1 Strategy 接口](../02-架构设计文档.md#31-strategy-接口) · [详细设计/02-策略层](../03-详细设计/02-策略层.md)（待编写）· [数据模型 §3.2.1 signals](../05-数据模型.md#321-signals策略信号)

## 1. 用途

`Strategy` 是所有策略的**抽象基类**。回测引擎和实盘引擎都通过这个接口驱动策略，**策略代码不感知自己在哪种模式**。

---

## 2. Strategy 基类签名

```python
class Strategy(ABC):
    name: str                    # 'S1_btc_trend' 等，用于落库
    version: str                 # 策略代码 git short hash（CI 注入）
    config_hash: str             # 当前配置参数 SHA-256
    universe_version: str        # 当前币池版本

    # ============ 必须实现 ============

    @abstractmethod
    def required_data(self) -> DataRequirement:
        """声明依赖数据。框架据此预热数据 + 订阅 WS。"""

    @abstractmethod
    def on_bar(self, bar: Bar, ctx: StrategyContext) -> list[Signal]:
        """K 线收盘时调用。返回 0..N 个信号。**必须是纯函数**（不写 IO）。"""

    # ============ 可选回调 ============

    def on_tick(self, tick: Tick, ctx: StrategyContext) -> list[Signal]:
        """tick 级响应。默认 no-op。"""

    def on_fill(self, fill: Fill, ctx: StrategyContext) -> None:
        """成交回调。可用于状态机更新。默认 no-op。"""

    def on_position_update(self, position: Position, ctx: StrategyContext) -> None:
        """持仓变更回调（含强平、止损触发）。默认 no-op。"""

    def on_funding(self, rate: FundingRate, ctx: StrategyContext) -> list[Signal]:
        """资金费率结算时调用（仅永续策略关心）。默认 no-op。"""

    # ============ 生命周期 ============

    def on_start(self, ctx: StrategyContext) -> None:
        """启动钩子。可用于初始化。"""

    def on_stop(self, ctx: StrategyContext) -> None:
        """停机钩子。"""
```

## 3. 数据契约

### 3.1 DataRequirement（required_data 返回值）

```python
@dataclass
class DataRequirement:
    symbols: list[str]                      # ['BTC/USDT', 'ETH/USDT']
    timeframes: list[str]                   # ['1h', '4h']
    history_lookback_bars: int = 500        # 启动时预热多少根历史 K 线
    needs_funding: bool = False             # 是否订阅资金费率
    needs_orderbook_l1: bool = False        # 是否订阅 best bid/ask
    needs_orderbook_l5: bool = False        # 订阅前 5 档（不轻易开）
```

### 3.2 Bar（K 线数据）

```python
@dataclass(frozen=True)
class Bar:
    symbol: str
    timeframe: str
    ts: int          # 开盘时间 ms (UTC)
    o: float
    h: float
    l: float
    c: float
    v: float
    q: float         # 成交额
    closed: bool     # True=已收盘的稳定 K 线；False=进行中
```

**重要**：`on_bar()` 默认**只在 `closed=True` 时调用**。如果策略需要"未收盘"的 bar，必须在 `required_data()` 里显式声明 `subscribe_partial_bars=True`，且要承担"价格回撤导致信号反复"的风险。

### 3.3 Signal（策略输出）

```python
@dataclass
class Signal:
    side: Literal['long', 'short', 'close']
    symbol: str
    entry_price: float | None        # None 表示 '市价'
    stop_price: float                # **必填**，强制止损依赖
    target_price: float | None       # 可选目标价
    confidence: float                # 0-1
    suggested_size: float            # 建议仓位（base asset），框架可能根据 L2 修正
    time_in_force: Literal['GTC','IOC','FOK','DAY'] = 'GTC'
    rationale: dict                  # 决策依据，会序列化为 signals.rationale JSON
    expires_in_ms: int = 60_000      # 信号有效期，过期被 L1 拒掉
    trade_group_id: str | None = None  # 多腿策略（如 S3）用
```

**约束**：
- `stop_price` 必填——策略产出信号时就必须知道止损位
- `entry_price=None`（市价）时，`stop_price` 解释为"成交价 ± offset"由风控层换算
- `confidence` 不直接作为下单阈值（[ADR-007](../07-决策日志/ADR-007-禁止LLM直接产生信号.md)）

## 4. StrategyContext（运行时上下文）

```python
class StrategyContext:
    # 数据访问（统一接口，研究 / 生产无感）
    data: DataFeed                          # 见 02-数据接口.md

    # 状态查询
    def get_position(self, symbol: str) -> Position | None: ...
    def get_balance(self) -> Balance: ...
    def get_open_orders(self, symbol: str) -> list[Order]: ...

    # 时间
    def now_ms(self) -> int: ...            # 当前时间 ms（回测注入）

    # 持久化（策略状态：仅本策略可见）
    def kv_get(self, key: str) -> Any: ...
    def kv_set(self, key: str, value: Any) -> None: ...

    # 日志
    def log(self, level: str, msg: str, **kwargs) -> None: ...
```

**ctx 不暴露**：
- `OMS` / `submit_order()`（策略不直接下单，只产 Signal）
- 其他策略的状态（隔离）
- secrets / API key

## 5. 行为约定

### 5.1 幂等性

- 同一 K 线被重复推送时（如恢复阶段），`on_bar()` 必须输出相同的信号
- 信号的去重在 L1 风控层做（5s 同 strategy+symbol+side 信号合并）

### 5.2 时间语义

- `on_bar()` 在 K 线**收盘后 ≤ 200ms** 内调用
- `bar.ts` 是开盘时间，不是收盘时间
- 跨时区数据全部 UTC 毫秒，策略代码不应自己处理时区

### 5.3 阻塞与异常

- `on_bar()` 必须在 **500ms 内**返回；超时框架记录但不杀进程
- 策略 raise `StrategyError` → 该信号被丢弃，落 risk_events
- 策略 raise 其他异常 → 框架捕获 + critical 告警，**该策略被自动停用**到下次重启
- **不允许**在策略里做 sleep / 阻塞 IO / requests 调用

### 5.4 副作用

- `on_bar()` 是纯函数：不写 DB、不发请求、不改全局状态
- 策略需要持久化状态用 `ctx.kv_set/kv_get`
- 跨 K 线状态（如均线值）允许放在策略实例属性里，框架保证单进程单实例

## 6. 使用示例

```python
class S1BtcEthTrend(Strategy):
    name = 'S1_btc_eth_trend'
    version = '__GIT_SHA__'           # CI 注入
    config_hash = '__CONFIG_HASH__'   # 启动时计算

    def required_data(self) -> DataRequirement:
        return DataRequirement(
            symbols=['BTC/USDT', 'ETH/USDT'],
            timeframes=['4h'],
            history_lookback_bars=500,
        )

    def on_bar(self, bar: Bar, ctx: StrategyContext) -> list[Signal]:
        if bar.symbol not in ('BTC/USDT', 'ETH/USDT'):
            return []
        if bar.timeframe != '4h':
            return []

        history = ctx.data.get_candles(bar.symbol, '4h', n=200)
        atr = compute_atr(history, period=14)
        ma_fast, ma_slow = compute_ma(history, 20), compute_ma(history, 50)

        if ma_fast > ma_slow and ctx.get_position(bar.symbol) is None:
            return [Signal(
                side='long',
                symbol=bar.symbol,
                entry_price=None,                 # 市价
                stop_price=bar.c - 1.5 * atr,
                confidence=0.6,
                suggested_size=ctx.get_balance().equity * 0.05 / bar.c,
                rationale={'mode':'trend_breakout','atr':atr,'ma_fast':ma_fast,'ma_slow':ma_slow},
            )]
        return []
```

## 7. 错误码 / 异常

| 异常类 | 来源 | 处置 |
|---|---|---|
| `StrategyError` | 策略主动 raise | 信号丢弃 + risk_events |
| `DataNotReadyError` | `ctx.data.get_*` 历史数据不足 | on_bar 跳过本次 |
| `KVStoreError` | `ctx.kv_*` IO 错误 | warn + 继续 |
| 其他 Exception | 编程错误 | 策略停用 + critical 告警 |
