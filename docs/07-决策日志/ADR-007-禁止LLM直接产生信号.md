# ADR-007 · 禁止用 LLM 直接产生交易信号

> **状态**：已生效
> **决定时间**：2026-04-29
> **影响范围**：策略层、AI 集成
> **关联**：[ADR-005 起步 3 策略](./ADR-005-起步3个策略而非10个.md)

---

## 1. 背景

业界一个常见诱惑：让 LLM 直接读各种信息（行情、新闻、链上数据），输出"该不该开仓 + 置信度"，系统按 `confidence ≥ 阈值` 自动下单。开发速度极快，看似 LLM 自带"专家知识"。

维护者已积累了相对完整的实测数据（百单级别样本）显示这种模式的真实表现：

| 模型给的 confidence | 实际胜率 | 标定偏差 |
|---|---|---|
| 0.60 | ≈ 35% | -25 个百分点 |
| 0.62 | ≈ 46% | -16 个百分点 |
| 0.65 | ≈ 34% | -31 个百分点 |

→ LLM 系统性 overconfident，**越自信反而越错**，且趋势稳定不是噪声。

需要决定本项目是否复用这种"LLM 直接产生信号"的模式。

---

## 2. 决定

**禁止用 LLM 直接产生交易信号。**

允许：
- LLM 用作**辅助打分**（新闻情绪、事件归类、报告摘要）
- 输出作为**因子**进入因子库，再由量化策略消费
- 用于**研究区**的策略生成 brainstorm（但代码必须人工写）

禁止：
- LLM 直接输出 `place_order(...)` 或 `Signal` 对象
- LLM 输出的 confidence 直接作为开仓阈值
- 没有量化因子参与的纯 LLM 策略

---

## 3. 备选方案

### A. LLM 直接产生信号（已被实测验证不可信，禁止）
**优点**：开发快，LLM 自带"专家知识"。  
**缺点**：
- LLM 系统性 overconfident
- 没有针对加密市场的 calibration
- 模型升级后行为可能完全变化（GPT-4 → GPT-5 时）
- 无法回测（LLM 不能完全复现）
- API 成本高（每次决策都要调）

### B. LLM 输出因子（本 ADR）
**优点**：
- 因子可量化、可回测、可校准
- 与传统因子并列，纳入策略框架
- 单一因子失效不影响整体
**缺点**：
- 增加因子库的复杂度
- LLM 因子的"特征工程"需要研究

### C. 完全不用 LLM
**优点**：极致简单。  
**缺点**：失去新闻情绪、事件等"非数值"信号源。

---

## 4. 选择理由

**关键权衡**：LLM 的潜在 alpha vs 直接信号的不可信。

实测数据已明确显示：**LLM 直接信号在金融场景不可信**（详见 §1 标定表）。

但 LLM 在以下场景仍有价值：
- **新闻情绪打分**：解读 Tether 增发/Fed 决议/CEO 推特
- **事件归类**：是否属于"系统性风险"
- **链上数据解读**：大额转账意图

→ 这些用作**因子**，由量化策略消费时**强制做 calibration**：
- 历史回测中，LLM 因子值 0.8 的样本实际胜率应该接近 0.8
- 偏差大就 reweight 或弃用

---

## 5. 后果

### 正面
- 避开已验证的"LLM 直接信号"陷阱
- 策略可回测、可解释
- LLM 升级不会突然让策略失效
- API 成本可控（因子离线计算）

### 负面
- 失去"快速试 LLM 策略"的灵活性
- 因子工程化 LLM 输出需要额外开发

### 中性
- LLM 仍可用于研究区脑暴和报告生成（不是核心信号源）

---

## 6. 何时重新审视

满足以下条件时重新评估：

- LLM 在金融场景的 calibration 显著改善（业界论文/产品验证）
- 出现专门针对加密市场训练的开源模型，且公开 backtest 结果优秀
- 本项目自己有 1000+ 单实盘数据后，可重新评估 LLM 因子的 alpha
