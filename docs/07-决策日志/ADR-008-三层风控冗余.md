# ADR-008 · 三层风控冗余（L1/L2/L3）

> **状态**：已生效（不可妥协）
> **决定时间**：2026-04-29
> **影响范围**：风控层、所有信号→订单链路
> **关联**：[ADR-003 强制交易所侧止损](./ADR-003-强制交易所侧止损.md)

---

## 1. 背景

单笔单方向交易的风控相对简单：单笔仓位上限、连亏 N 次熔断、总资金占用上限。Kelly 公式给出建议本金占比即可。

币圈合约风控复杂得多：
- 杠杆持仓有强平风险（24/7）
- 单币种敞口可能因价格变动放大
- 多策略叠加可能产生隐性高杠杆
- 滑点可能让"理论上不亏"的策略实际亏
- 黑天鹅可能瞬间穿透多个止损位

维护者过往项目中曾经只配置**单层**风控（每笔仓位上限 + 连亏熔断）：
- 这层规则没被人为放宽时一切正常
- 一旦凭"我感觉策略好用"放宽了限制（如把单笔上限从 1% 改到 50%、把日亏熔断从 5% 改到 99%）
- 策略一出 bug 就**没有任何兜底**

**币圈如果同样用单层风控**，第一次有 bug 就是灾难。

---

## 2. 决定

**风控分三层**，任何一层都可能有 bug，三层冗余让"全部失效"概率 ≈ 0：

### L3：投资组合层（信号过滤）

在信号产生后、计算仓位前。

| 规则 | 阈值 | 触发动作 |
|---|---|---|
| 总杠杆 | ≤ 3x | 拒绝 |
| 单币种敞口 | ≤ 25% 总资金 | 拒绝 |
| 单日浮亏 | ≥ 5% | 24h 熔断 |
| 最大回撤 | ≥ 20% | **强制清仓** |
| 连续亏损 | ≥ 5 笔 | 24h 减半仓位 |

### L2：仓位层（仓位计算）

在 L3 通过后，计算具体下多少。

- ATR-based 仓位算法：`size = risk_amount / (ATR × multiplier)`
- 单笔风险 ≤ 1% 总资金
- 强平距离 ≥ 30%（合约）
- Kelly fraction = 0.25（不全仓 Kelly）

### L1：订单层（执行前）

在 L2 计算完，订单提交交易所前。

- 滑点检查：市价单预估滑点 > 0.2% 取消
- 部分成交：5min 内未完全成交则改价或取消
- 重复检查：同 symbol 5s 内重复信号去重
- **强制止损单**：开仓后 5s 内必须挂止损（[ADR-003](./ADR-003-强制交易所侧止损.md)）

---

## 3. 备选方案

### A. 单层风控（已被实测验证错误，禁止）
**优点**：实现简单。  
**缺点**：单点失败致命，币圈不适用。

### B. 双层（仓位 + 订单）
**优点**：覆盖大部分场景。  
**缺点**：投资组合级风险（如总杠杆）无人监控。

### C. 三层（本 ADR）
**优点**：覆盖完整、有冗余。  
**缺点**：实现成本最高，需要三层间协调。

### D. 四层及以上
**优点**：理论更稳。  
**缺点**：边际收益递减、维护成本陡增。

---

## 4. 选择理由

**任何一层都可能有 bug**：
- L3 算总杠杆时如果忘记加上某个新建仓位 → 漏算
- L2 仓位算法在极端 ATR 时可能输出 0 或无穷
- L1 滑点估计在低流动性币上可能不准

**冗余设计的数学**：

| 单层准确率 | 单层失败概率 | 三层都失败 |
|---|---|---|
| 99% | 1% | 0.0001%（百万分之一） |
| 95% | 5% | 0.0125%（八千分之一） |
| 90% | 10% | 0.1%（千分之一） |

→ 即使每层准确率只有 90%，三层冗余也能把整体风险降到千分之一。

**三层职责分离**：
- L3 看"全局"（账户级）
- L2 看"单策略"（信号级）
- L1 看"具体订单"（执行级）

→ 各管一摊，bug 不会跨层扩散。

---

## 5. 后果

### 正面
- 即使某层有 bug，其他层兜底
- 可以分别测试各层（L3 用模拟账户、L2 用单笔信号、L1 用 mock 交易所）
- 风险事件落库后，能精准定位是哪层拒绝的（`source` 字段）

### 负面
- 实现成本：3 套规则、3 套测试
- 性能成本：每个信号要过 3 道检查（< 10ms 可接受）
- 调试复杂度：信号被拒后要追溯是哪层

### 中性
- 需要专门的 risk_events 表和 Dashboard 展示
- 配置文件三层规则要清晰组织

---

## 6. 何时重新审视

**永不取消**。可调的是：

- 各层的具体阈值（如总杠杆从 3x 改 2x）
- 是否新增第四层（如"链上风险层"）
- 各层之间的优先级和短路逻辑

但"三层结构"本身不可妥协。
