## 写作约定

每份子文档遵循：

```markdown
# 0X · 模块名

> **摘要**：一句话讲这个模块干啥。
> **关联**：架构对应章节 / 接口契约 / 数据模型表。

## 1. 模块边界
（输入 / 输出 / 依赖谁 / 谁依赖它）

## 2. 内部结构
（子组件、类图、关键状态）

## 3. 控制流
（关键流程的时序）

## 4. 关键算法
（如有 ATR 仓位、滑点估计、撮合逻辑等）

## 5. 异常与降级
（失败时怎么办）

## 6. 性能考量
（瓶颈、优化策略）

## 7. 测试策略
（单测、集成测、回测验证）

## 8. 待办与已知限制
```

> **不写**：具体代码、import 语句、库版本号。这些在源码注释/接口文档里。
