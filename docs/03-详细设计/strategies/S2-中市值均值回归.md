# S2 · 中市值币均值回归

> **风格**：均值回归（Mean Reversion）
> **状态**：规划阶段（Phase 3 实现）
> **预期角色**：卫星仓位策略，在震荡市补 S1 不足
> **最后更新**：2026-04-29

---

## 1. 策略思路

中市值币（top 20-100）经常出现**短期超卖反弹**：
- 因恐慌或集中卖出快速下跌
- 跌破均线后短期内（4-72h）回升至均值

不在大趋势下跌中"接飞刀"——只交易**日线趋势仍多头但短线超卖**的情况。

> **核心**：日线趋势确认方向 + 小时线找反转入场点。

---

## 2. 标的与周期

| 项 | 值 |
|---|---|
| 标的 | top 50 永续合约（按 24h 成交量排）**剔除 BTC/ETH** |
| 排除规则 | 上线 < 30 天的新币、稳定币（USDC/DAI 等）、绑定币（如 BNB） |
| 入场周期 | **1h K 线** |
| 趋势过滤周期 | **1d K 线** |
| 平均持仓时长 | 4-72 小时 |
| 杠杆 | **1-2x**（短期持仓，但仍控制） |

→ 与 S1 不重叠：S1 做 BTC/ETH，S2 做其他币，**避免同方向多头叠加**。

---

## 3. 入场条件

只做多头反转（不做空头反弹，统计上更稳定）：

### 3.1 大趋势过滤（1d K 线）

- 1d 收盘价 > 1d MA50（大方向多头）
- AND 1d MA20 > 1d MA50

→ 防止在熊市抄底。

### 3.2 入场触发（1h K 线）

**全部满足**：

- 1h RSI(14) < 25（超卖）
- 1h 收盘价触碰布林带下轨（BB(20, 2)）
- 该 1h K 线下跌幅度 ≤ 5%（防止崩盘式下跌）

### 3.3 流动性过滤

- 标的近 7 天日均成交额 > 5000 万 USDT
- 防止低流动性币的"假超卖"信号

### 3.4 资金费率过滤

- 当前 funding rate 不在极端值（绝对值 < 0.1%）
- 资金费率极端表示市场情绪极端，不适合做反转

---

## 4. 出场条件

### 4.1 止损（强制交易所侧）

- 入场价 × (1 - 0.05) → 即固定 -5% 止损
- 止损单挂 Binance 侧（[ADR-003](../../07-决策日志/ADR-003-强制交易所侧止损.md)）

### 4.2 止盈（主动平仓）

- 1h 收盘价回归布林带中轨（BB(20) 中轴）→ 一次性全部止盈
- 或浮盈达 +5% 强制止盈一半，剩余跟踪止损

### 4.3 时间止损

- 持仓超过 72h 未触发止盈也未止损 → 主动平仓
- 避免"温水煮青蛙"

---

## 5. 仓位管理

### 5.1 单笔仓位

```
风险金额 = 总资金 × 0.5%      （比 S1 更小，因为反转策略胜率高但盈亏比低）
仓位 = 风险金额 / 0.05         （止损 5%）
```

### 5.2 不加仓

- 反转策略 by design 不加仓（避免越亏越加）
- 一次性入场，到期出场

### 5.3 同时持仓数

- 最多 5 个不同标的同时持仓
- 避免分散过度（管理负担）也避免集中过度（黑天鹅风险）

---

## 6. 风控规则

| 规则 | 阈值 | 动作 |
|---|---|---|
| 单策略单日亏 | ≥ 3% 总资金 | 暂停 24h |
| 单策略连亏 | ≥ 6 笔 | 暂停 24h |
| 单标的当日开仓 | ≤ 1 次 | 防止越亏越买 |
| 同方向标的（>3 个超卖） | 不全开 | 取 RSI 最低的 3 个 |
| 单策略最大回撤 | ≥ 12% 策略资金 | 暂停 + 复盘 |

## 7. 预期表现（待回测验证）

| 指标 | 目标 |
|---|---|
| 胜率 | **55-65%**（高胜率） |
| 盈亏比 | 0.7-1.0（低盈亏比） |
| 年化 | 25-40% |
| Sharpe | 1.2-1.6 |
| 最大回撤 | 10-15% |
| 月度盈利月数 | 8-9/12 |

**特征**：
- 高频小赚 + 偶尔大亏（黑天鹅式跌穿）
- 震荡市表现最好
- 单边下跌市最难受

---

## 8. 失效场景

哪些情况这个策略会亏：

1. **黑天鹅快速崩盘**（312 模式）→ 5% 止损被穿透
2. **山寨币结构性下跌**（项目暴雷）→ 反转不成立
3. **趋势加速向下**（不是均值回归而是趋势）→ 止损连环触发
4. **流动性枯竭**→ 滑点严重

**应对**：
- 杠杆 ≤ 2x + 5% 硬止损 → 单笔损失可控
- 与 S1 反相关 → 趋势市 S1 赚钱，震荡市 S2 赚钱
- 严格的标的池（top 50）+ 流动性过滤

---

## 9. 与其他策略的相关性

| 策略 | 预期相关性 | 说明 |
|---|---|---|
| S1 BTC/ETH 趋势 | **强负相关** | 趋势市 S1 赚 / 震荡市 S2 赚 |
| S3 资金费率套利 | 几乎独立 | 套利不依赖方向 |

→ S1 + S2 组合能覆盖大多数 regime（趋势 + 震荡），互相补漏。
