# S3 · 资金费率套利

> **风格**：市场中性套利（Market Neutral Arbitrage）
> **状态**：规划阶段（Phase 3 实现）
> **预期角色**：稳定收益策略，不依赖方向判断
> **最后更新**：2026-04-29

---

## 1. 策略思路

永续合约的**资金费率（funding rate）** 是多空平衡机制：
- 多头力量大 → 资金费率为正 → 多头每 8 小时支付给空头
- 空头力量大 → 资金费率为负 → 空头支付给多头

当资金费率达到**极端值**时（如 +0.15%/8h），意味着多头过度拥挤，**反向开空 + 现货买入对冲** 可以：
- 收资金费（被付费）
- 用现货对冲掉价格风险（多空相抵）

**核心**：不预测涨跌，靠资金费收益。理论上**市场中性**。

---

## 2. 标的与周期

| 项 | 值 |
|---|---|
| 标的 | top 30 永续合约（按流动性排）+ 对应现货 |
| 触发周期 | **每 8 小时**（资金费结算前 30min 评估） |
| 持仓周期 | 几小时-几天（直到 funding rate 回归） |
| 杠杆 | **永续合约 1x（全保证金）** + 现货全仓买入 |

→ 不加杠杆，避免基差放大风险。

---

## 3. 入场条件

### 3.1 资金费率极端

- 当前 funding rate **绝对值 ≥ 0.10%**（即 ≥ 0.36% 日化）
- 历史 30 天该币 funding rate 同方向连续 ≥ 3 次

→ 表示市场情绪持续极端，回归概率高。

### 3.2 基差（spot vs perp）合理

- 永续价 / 现货价 偏离 ≤ 0.5%
- 偏离过大说明市场已经在反向定价，套利空间不足

### 3.3 流动性

- 永续 24h 成交额 ≥ 1 亿 USDT
- 现货 24h 成交额 ≥ 5000 万 USDT

→ 防止滑点吃掉资金费收益。

### 3.4 入场动作

**资金费率为正（+0.10%）的处理**：

1. **现货市场**买入 N 个币（用 USDT）
2. **永续合约**开空 N 个币（保证金用 USDT）

→ 价格涨了：永续亏损 = 现货盈利（对冲）
→ 价格跌了：永续盈利 = 现货亏损（对冲）
→ 唯一收益来源：每 8 小时收一次资金费

**资金费率为负的反向操作**（对称）：现货可借币做空 + 永续做多。但**借币成本高**，本项目第一阶段**只做正费率方向**。

---

## 4. 出场条件

### 4.1 funding rate 回归

- 当前 funding rate 跌到 **< 0.03%/8h** → 平仓
- 收益空间已耗尽

### 4.2 基差扩大止损

- 永续价 / 现货价 偏离 ≥ 1.0%（基差崩坏）→ **强制平仓**
- 这是套利策略最大风险

### 4.3 时间止损

- 持仓超过 7 天 funding rate 未回归 → 平仓
- 资金占用时间太长

---

## 5. 仓位管理

### 5.1 单笔仓位

```
单笔投入 ≤ 总资金 10%
（现货 50% + 永续保证金 50% 各占 5%）
```

→ 多个套利机会可以并行，避免单 symbol 集中。

### 5.2 不加仓

- 套利策略本身没有"行情判断"
- 每个机会独立，不连续加仓

### 5.3 同时套利数

- 最多 5 个不同标的同时套利
- 总占用 ≤ 50% 总资金

---

## 6. 风控规则

| 规则 | 阈值 | 动作 |
|---|---|---|
| 基差扩大 | ≥ 1.0% | **立即平仓双腿** |
| funding rate 已回归仍持仓 | > 4 小时 | 主动平仓 |
| 现货-永续滑点差 | ≥ 0.3% | 撤单重试或放弃 |
| 单笔套利亏损 | ≥ 1% | 平仓（理论上不应该发生） |
| 同时持仓数 | ≤ 5 个标的 | - |

**特殊场景**：
- 交易所现货-永续 API 不同步 → 单腿挂单失败 → 立即取消另一腿
- 入场时**双腿原子化**：要么都成交、要么都取消（半腿状态不能持续 > 30s）

### 6.1 双腿原子下单 / 数据模型

S3 不能用单 orders/positions 思路实现，必须用 [`trade_groups`](../../05-数据模型.md#325-trade_groups多腿事务组) 表把"现货腿 + 永续腿"绑成一个事务组：

| 概念 | 表 / 字段 |
|---|---|
| 套利组 | `trade_groups`（一个 group 对应一次套利操作） |
| 腿订单 | `orders.trade_group_id` 关联到 group |
| 腿持仓 | `positions.trade_group_id` 关联到 group |
| 入场原子性 | `trade_groups.status` = `forming → hedging → hedged` |
| 半腿告警 | 第一腿成交后 30s 第二腿未成交 → `status = unwinding`，撤第一腿 |
| 平仓原子性 | `status = unwinding → closed`；任一腿平仓失败 → `partial_alert` 人工介入 |

完整状态机见 [`数据模型 §3.2.5 trade_groups`](../../05-数据模型.md#325-trade_groups多腿事务组)。

### 6.2 双腿失败矩阵

| 失败场景 | 检测 | 处置 |
|---|---|---|
| 第一腿挂单被拒 | OMS 收到 reject | 直接 `broken`，不发第二腿 |
| 第一腿成交、第二腿挂单被拒 | OMS 收到 reject | 立刻撤第一腿（市价反向平），group → `unwinding → broken` |
| 第一腿成交、第二腿挂单超时未成交 | 30s timeout | 同上 |
| 双腿成交后基差扩大 ≥ 1% | 风控监控 | group → `unwinding`，双腿同时市价平仓 |
| 平仓时一腿成功一腿失败 | OMS 重试 3 次失败 | group → `partial_alert`，**critical 告警 + 暂停新 S3 信号** |
| 现货 / 永续 API 单独不可用 | 健康检查 | 拒绝产生 S3 新信号；已持仓正常监控 |

## 7. 预期表现（基于历史数据）

| 指标 | 目标 |
|---|---|
| 胜率 | **80-90%**（套利策略高胜率） |
| 平均单笔收益 | 0.1-0.3% |
| 年化 | 15-25%（保守） |
| Sharpe | **2.0+**（市场中性，波动小） |
| 最大回撤 | 5-8% |
| 月度盈利月数 | 11-12/12 |

**特征**：
- 收益稳定但绝对值低
- 极端情况（基差崩坏）会有阵痛
- 适合放在投资组合的"防守仓位"

---

## 8. 失效场景

哪些情况这个策略会亏：

1. **基差崩坏**（如交易所故障/巨鲸攻击）→ 双腿对冲失效
2. **funding rate 长期不回归**（罕见但可能）→ 资金占用过久
3. **现货下市/合约清算**（极端项目风险）→ 失去对冲
4. **API 故障**导致只成交一腿 → 暴露在方向风险

**应对**：
- 标的池仅 top 30（这种情况罕见）
- 基差扩大 1% 强制平仓
- 双腿原子化下单

## 9. 与其他策略的相关性

| 策略 | 预期相关性 | 说明 |
|---|---|---|
| S1 BTC/ETH 趋势 | **几乎独立** | 套利不依赖方向 |
| S2 中市值均值回归 | **几乎独立** | 套利收益来自资金费 |

→ **S3 是组合的"压舱石"**：S1+S2 因 regime 失效时，S3 仍可能稳定盈利。
