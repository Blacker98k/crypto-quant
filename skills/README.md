> **格式参考**：用户的 `~/.cursor/skills/` 目录里的现有 SKILL（如 `GitHub私有仓库发布/SKILL.md`）。

## 项目内 SKILL vs 用户级 SKILL

| 类型 | 位置 | 范围 | 例子 |
|---|---|---|---|
| **项目内** | `skills/`（本目录） | 跟着 git 走，clone 即得 | "启动 crypto-quant" |
| **用户级** | `~/.cursor/skills/` | 跨项目，用户私人 | "GitHub 私有仓库发布" |

如果某项目内 SKILL 后续证明跨项目通用，可手动复制到 `~/.cursor/skills/`。

## 写作模板

每个技能一个目录 + `SKILL.md`，开头是 YAML frontmatter：

```markdown
---
name: 技能名（中文）
description: 一句话讲做什么。当用户说 X / Y / Z 时使用。
keywords: [关键词1, 关键词2, alias_en, ...]
---

# 技能名

## 何时使用
（什么场景下读这份 SKILL）

## 前置条件
（执行前需要满足的条件）

## 操作步骤
1. ...
2. ...

## 验证
（怎么知道执行成功）

## 排错
（常见失败 + 解决）

## 关联
- 相关文档
- 相关其他技能
```

可选附件：
```
skills/技能名/
├── SKILL.md
├── examples/         # 示例命令、截图
└── templates/        # 可复用模板（如配置文件）
```

## 编写优先级

| Phase | 必备的 SKILL |
|---|---|
| Phase 0（现在） | 无（除了"提交 GitHub"，但走用户级 SKILL） |
| Phase 1 | 启动停止系统、拉取历史数据 |
| Phase 2 | 跑回测 |
| Phase 3 | 上线新策略、应急处理 |

→ Phase 0 此目录可以先放空，等 Phase 1 真正动手时再写。
