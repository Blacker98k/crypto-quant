### 2026-04-29 · 强制交易所侧止损单（不可妥协）

**摘要**：任何持仓必须在交易所侧挂止损单。开仓后 5s 内挂止损是原子操作。

**理由**：本地服务可能宕机/断网/被挂起，只有交易所侧止损永远生效。代价是可能被插针扫掉一些利润，但极端行情时是救命的。

**完整 ADR**：[`ADR-003-强制交易所侧止损.md`](../docs/07-决策日志/ADR-003-强制交易所侧止损.md)

**摘要**：不用现成开源框架（freqtrade 等），自研。借鉴这些项目的 idea 但代码自己写。

### 2026-04-29 · 禁止用 LLM 直接产生信号

**摘要**：LLM 仅可用作辅助打分（新闻情绪、事件归类），不允许成为信号的唯一来源。

### 2026-04-29 · 远程 Agent 写权限的代码合入审查（ADR-009）

**摘要**：常驻 Agent（如 OpenClaw 这类 7×24 在线工具）虽然在本机有完整的文件系统能力，但**默认不持有远程仓库 push 凭据**——所有产出先落本地 patch / 工作目录，由维护者 review 后才决定是否进 git。主分支写权限**仅限维护者本人**。

**核心约束**：
- 双层闸门：① 维护者本机 review 拦掉绝大多数临时探索；② PR 人审拦掉看似 OK 但有风险的实现
- 常驻 Agent 默认产出位置 = 本地 patch / 工作目录文件（**不**直接 push 远程）
- 主分支保护：禁止直推、必须 PR、必须 review
- 高敏感文件（secrets / 风控阈值 / 实盘开关 / 部署脚本）**绝对禁止** Agent 直接动；维护者本人接手改
- `.gitignore` 例外：Agent 可提 PR 加显然产物（`*.log` / `__pycache__` 等），但不得增删可能改变敏感数据入库范围的条目
- 每次 Agent 提交带 `[agent: <name>]` 前缀 + 任务摘要 + reviewer 字段

**完整 ADR**：[`ADR-009-远程Agent与可信代码边界.md`](../docs/07-决策日志/ADR-009-远程Agent与可信代码边界.md)
