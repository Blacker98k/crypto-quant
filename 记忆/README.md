## 记忆库使用守则

**读取**：

- 进项目第一件事是读 `01-项目概览.md`，了解上下文
- 拿到具体任务后才查对应的 docs/ 子文档
- 不要一次性把 docs/ 全读了——会爆上下文

**更新**：

- **每次会话结束前**，必须更新 `02-当前状态.md`，记录这次做了啥、下次接什么
- 做出**重大决策**时（影响架构、改变方向），新增一条到 `04-最近决策.md`，并考虑写一份完整 ADR 到 `docs/07-决策日志/`
- 改了**编码风格**或**新约定**，更新 `03-编码约定.md`
- `01-项目概览.md` 几乎不改，只在 phase 切换时更新

**结构原则**：

- 每份文档 ≤ 200 行（这个目录里的）
- 关键信息**靠前**
- 不重复 docs/ 里的内容，只**链接**
